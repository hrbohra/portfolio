## What it is

<Plain>
Nearly eight thousand supplier product photos, none labelled, all needed sorting into
categories before the store could launch. Doing it by hand takes about a month. I did it in
a weekend with a free AI model running on my own machine, wrapped in a pipeline built to
survive crashes.
</Plain>
<Eng>
Batch vision-model classification: 7,975 images into 7 categories using qwen2.5vl:7b served
locally by Ollama. Parallel workers, checkpoint file as source of truth, resume-safe across a
multi-hour run that was interrupted several times. Output feeds the storefront catalogue
builder. Full story in the linked case.
</Eng>

## What it proves

<Plain>
Knowing when the expensive option is unnecessary. The clever part was not the AI, it was the
plumbing that let a free model finish a real job reliably.
</Plain>
<Eng>
Local-inference economics, batch reliability engineering, and honest taxonomy design that
keeps a small model's boundary errors cheap to correct.
</Eng>
